# Архитектурные ориентиры

Документ фиксирует текущие границы проекта, но не считается неизменяемой
спецификацией.

## Продукт

Приложение объединяет:

- блочные заметки в духе Notion и Obsidian;
- задачи;
- управляемые файловые пространства в духе Eagle;
- локальную работу без обязательной регистрации и сервера.

## Платформы

1. Windows
2. Android
3. macOS
4. iOS не входит в ближайшие планы

## Слои

- `React` отвечает за интерфейс и пользовательские сценарии.
- `Rust/Tauri` отвечает за файловую систему, индексирование и платформенные API.
- `SQLite` позднее станет восстанавливаемым локальным индексом.
- Переносимые файлы пространства останутся основным пользовательским форматом.

## Правила на раннем этапе

- Не добавлять backend до появления сценария, который без него невозможен.
- Не связывать UI напрямую с Tauri-командами; использовать адаптеры в `shared`.
- Не хранить пользовательские файлы внутри SQLite.
- Не считать локальную индексную базу единственной копией пользовательских данных.
- Сначала проверить сценарий интерфейсным прототипом, затем подключать хранение.
